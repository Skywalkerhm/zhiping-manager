# 智评管家 - 部署指南

## 📋 部署前准备

### 1. 服务器要求

- **操作系统**: Ubuntu 20.04+ / CentOS 7+
- **CPU**: 2 核+
- **内存**: 4GB+
- **存储**: 20GB+
- **Python**: 3.9+
- **Node.js**: 18+

### 2. 域名和证书

- 已购买域名
- SSL 证书（可使用 Let's Encrypt 免费证书）

### 3. 数据库

- MySQL 8.0+
- Redis 7.0+

## 🚀 部署步骤

### 步骤 1: 安装依赖

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Python 和依赖
sudo apt install -y python3.9 python3.9-venv python3-pip mysql-server redis nginx git

# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
python3 --version
node --version
npm --version
```

### 步骤 2: 克隆项目

```bash
# 创建应用目录
sudo mkdir -p /var/www/zhiping-manager
cd /var/www/zhiping-manager

# 克隆项目（替换为你的 GitHub 地址）
sudo git clone https://github.com/yourusername/zhiping-manager.git .
sudo chown -R $USER:$USER /var/www/zhiping-manager
```

### 步骤 3: 配置后端

```bash
# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp config/.env.example .env
nano .env  # 编辑配置

# 初始化数据库
mysql -u root -p < scripts/init_db.sql
```

### 步骤 4: 配置前端

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 构建生产版本
npm run build

# 返回根目录
cd ..
```

### 步骤 5: 配置 Gunicorn

```bash
# 创建 systemd 服务文件
sudo nano /etc/systemd/system/zhiping.service
```

添加以下内容：

```ini
[Unit]
Description=Zhiping Manager API
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/zhiping-manager
ExecStart=/var/www/zhiping-manager/venv/bin/gunicorn -w 4 -k uvicorn.workers.UvicornWorker app.main:app -b 0.0.0.0:8000
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start zhiping
sudo systemctl enable zhiping
sudo systemctl status zhiping
```

### 步骤 6: 配置 Nginx

```bash
sudo nano /etc/nginx/sites-available/zhiping
```

添加以下配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/zhiping-manager/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源
    location /static/ {
        alias /var/www/zhiping-manager/static/;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/zhiping /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 步骤 7: 配置 HTTPS（Let's Encrypt）

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

### 步骤 8: 配置防火墙

```bash
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable
```

## 🔧 运维管理

```bash
sudo systemctl restart nginx
```

### 更新代码

```bash
cd /var/www/zhiping-manager
git pull
source venv/bin/activate
pip install -r requirements.txt
cd frontend && npm install && npm run build
sudo systemctl restart zhiping
```

### 数据库备份

```bash
# 创建备份脚本
nano /var/www/zhiping-manager/scripts/backup.sh
```

```bash
#!/bin/bash
BACKUP_DIR="/var/backups/zhiping"
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -u root -pYOUR_PASSWORD zhiping_manager > $BACKUP_DIR/backup_$DATE.sql
find $BACKUP_DIR -name "backup_*.sql" -mtime +7 -delete
```

设置定时任务：

```bash
crontab -e
# 每天凌晨 2 点备份
0 2 * * * /var/www/zhiping-manager/scripts/backup.sh
```

## ⚠️ 常见问题

### 1. 端口被占用

```bash
# 查看端口占用
sudo lsof -i :8000
sudo lsof -i :80
```

### 2. 权限问题

```bash
sudo chown -R www-data:www-data /var/www/zhiping-manager
sudo chmod -R 755 /var/www/zhiping-manager
```

### 3. 数据库连接失败

```bash
# 检查 MySQL 状态
sudo systemctl status mysql

# 检查连接
mysql -u root -p -e "SHOW DATABASES;"
```

### 4. Nginx 配置错误

```bash
# 测试配置
sudo nginx -t

# 重新加载
sudo nginx -s reload
```
